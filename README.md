# java-lotto-precourse
구현 기능 목록
1. 로또 발매기 구현
- 1-1. 로또 구입 금액 입력받기
- 1-2. 로또 당첨 번호 입력받기
- 1-3. 로또 당첨 번호 파싱(쉼표를 기준으로 파싱)
- 1-3. 보너스 번호 입력받기

2. 로또 발행 기능
- 2-1. 발행할 로또 수량 계산
- 2-2. 로또 발행하기

3. 로또 당첨 결과 계산하기
- 3-1. 발행한 로또 티켓마다 등수 매기기
- 3-2. 당첨된 로또 티켓의 전체 상금 계산하기
- 3-3. 발행한 로또 티켓마다 로또 번호와 당첨 번호 비교하기
  
4. 당첨 내역 출력
- 4-1. 당첨 내역 출력하기

5. 수익률 계산
- 5-1. 소수점 둘째 자리에서 반올림

6. 등수 및 상금 관리 (Rank Enum)
- 6-1. 등수 정의
- 6-2. 등수 판별 메서드 정의
- 6-3. 상금 조회 기능
- 6-4. 일치 조건 판별

7. 테스트 코드 작성
- LottoTest :
  - 7-1. 로또 번호가 1-45 범위를 벗어나면 예외 
  - 7-2. 로또 번호 개수가 6개 미만이면 예외 
- WinningLottoTest :
  - 7-3. 보너스 번호가 당첨 번호와 중복되면 예외
  - 7-4. 정상 생성 시 접근자 동작 확인
- RankTest :
  - 7-5. 일치 개수와 보너스 여부에 따라 올바른 Rank 반환 확인
  - 7-6. 등수별 상금이 기대값과 일치하는지 확인
- LottoMachineTest :
  - 7-7. 구입 금액으로 구매 가능한 로또 개수 계산 확인
  - 7-8. 발행된 로또 수량이 구매 개수와 일치 확인
  - 7-9. 각 발행 티켓이 6개 유니크 번호(1~45 범위) 보유하는지 확인
- MoneyTest :
  - 7-10. 구입 금액이 1000원 단위가 아니면 예외가 발생
- LottoResultTest :
  - 7-11. 수익률이 소수점 둘째 자리에서 반올림되는지 확인
    
8. 예외 처리
- 8-1. 로또 번호가 1-45 범위를 벗어나면 예외 
- 8-2. 로또 번호 개수가 6개 미만이면 예외
- 8-3. 보너스 번호가 당첨 번호와 중복되면 예외
- 8-4. 구입 금액이 1000원 단위가 아니면 예외가 발생
